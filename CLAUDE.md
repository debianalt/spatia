## Pipeline Conventions
- **Raster nodata**: rasterio nodata=0 en binarios S1 conflicta con dry land â†’ usar `ignore_src_nodata=True`. JRC usa -128 (not water) y -32768 (nodata) â†’ se maneja en `zonal_stats_rasterio`.
- **Zonal stats**: SIEMPRE usar `zonal_stats_rasterio` (polygon masking), NUNCA `zonal_stats_sampling` (centroid, solo legacy).
- **Bbox**: padded a [-56.10, -28.20, -53.55, -25.44] para cubrir hexĂˇgonos del borde.
- **GCS bucket `spatia-satellite`**: NO existe. Exportar a Google Drive (`--drive` flag) y descargar via Drive API con credenciales EE.
- **GEE auth**: credenciales en `~/.config/earthengine/credentials` (OAuth user, scopes: earthengine, cloud-platform, drive, devstorage).
- **R2 uploads**: `npx wrangler r2 object put neahub/... --file ... --remote` â€” el pipeline usa `upload_to_r2.py` pero NO pasa `--remote`, hacer upload manual con `--remote` despuĂ©s.
- **Parquet cache**: DuckDB-WASM en browser cachea parquets. Usar cache-buster `?v=N` en `getParquetUrl()` al cambiar schema (incrementar N en config.ts).

## Flood Risk Layer
- **Data sources**: JRC Global Surface Water v1.4 (Landsat 1984â€“2021) + Sentinel-1 SAR (current extent)
- **Parquet columns**: `h3index`, `jrc_occurrence` (0-100%), `jrc_recurrence` (0-100%), `jrc_seasonality` (0-12), `flood_extent_pct` (0-100%), `flood_risk_score` (0-100)
- **Score formula**: 50% JRC occurrence + 20% JRC recurrence + 30% S1 current extent
- **Pipeline**: `python pipeline/run_flood_update.py --skip-gee` (con rasters locales)
